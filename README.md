# Catparse

Основным источником стал сайт https://companies.rbc.ru/ "РБК Компании" по следующим причина:

1.Возможность поиска по отдельным ОКВЭДам

2.Предоставление актуальных данных по годовой выручке компаний, что позволяет провести фильтрацию на этапе сбора данных.

Для запуска кода необходимо вставить ссылку с нужного раздела оквэда сайта "РБК Компании" в 204 строке в файле main.py и подключить все связи. Доп функционал - Можно отдельно настроить число считываемых страниц поменяв значение max_pages в функции def parse_all_pages(base_url, max_pages=100): на 120 строке; Можно настроить фильтр годовой выручки на строке 110.

Какие ограничения / упрощения заложили:

1.Выставил максимальное значение страниц для парсинга, так как на в основном компании с оборотом более 100 млн в год располагаются в начале списка. 

2.Сделал универсальное формирование ссылок при переключении страниц на случай использования с другими сервисами.


Подход:<br></br>
1.Определил основные оквэды компаний, потенциально заинтересованных в использовании cat-систем, TMS, платформ локализации. Логика определения оквэдов состояла из двух частей:
a.Поиск архивных вакансий на hh.ru требующих навыков работы с cat-системами, далее проверка компаний-работодателей и выписывание оквэдов, наиболее часто встречающихся у таких компаний.
b.Самостоятельное определение окэдов исходя из основных функций cat-систем.

2.Адаптировал парсер под работу с "РБК Компании" как сервисом с наиболее подробной информацией подходящей как для поиска по оквэда, так и с указанной актуальной годовой выручкой.

3.Через LLM в режиме search нашел сайты компаний и пропустил через LLM на поиск упоминания работы с Cat-cистемами.

4.Собрал основные критерии, отличающие компании использующие cat-системы от остальных. Далее запустил ллм с сохраненным списком критериев на разметку сайтов согласно критериямю

5.Пропустил компании через ЛЛМ на поиск упоминаний cat-систем в заявлениях компаний, публикациях и т.д.

6.Всю выручку проверил через "ресурс БФО" (Государственный информационный ресурс бухгалтерской финансовой отчетности). 


Что получилось / что не получилось:

1.Удалось собрать быстрый парсер без блокировок и использования прокси.

2.Удалось собрать большую выгрузку компаний интересующих оквэдов с фильтрацией по годовой выручке.


1.Не удалось получить неопровержимые доказательства по многим компаниям с выручкой более 100 млн в силу отсутствия каких-либо упоминаний по использования искомых систем на сайтах

2.Планировал, но не успел автоматизировать загрузку и выгрузку данных из LLM

3.Не успел полностью разобрать собранный массив информации по всем интересующим оквэдам.

4.Планировал, но не успел переделать логику создания названия для excel-таблиц

5.Можно было собрать отдельный парсер по поиску ключевых слов на сайтах компаний, но ручной анализ сайтов переводческих бюро показал, что это было бы неэффективно ввиду отсутствия явных ссылок на искомые системы и малочисленности самих переводчиков, подходящих по условиям.

Роль ЛЛМ:

1.адаптировал парсер под html-блоки сайта "РБК Компании".

2.использовал для deepsearch по компаниям.

3.использовал для проверки сайтов и деятельности компании на предмет потенциального использования cat-систем.

4.подготовил ключевые метрики оценивания компаний на предмет использования cat-систем
